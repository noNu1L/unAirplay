===========================================
  unAirplay - Version Log / 版本日志
===========================================

v1.1.2 (2026-02-06)
-------------------

1. Feature: AirPlay Device Offline Detection and Auto-Removal
   新功能：AirPlay 设备离线检测与自动清理

   - Added AIRPLAY_OFFLINE_THRESHOLD configuration (default: 3 scans)
   - Devices offline for consecutive scans will be automatically removed
   - Configurations automatically restored when device comes back online

   添加 AIRPLAY_OFFLINE_THRESHOLD 配置（默认：10 次扫描）
   设备连续多次扫描未检测到将自动删除虚拟设备
   设备重新上线时自动恢复配置

v1.1.1 (2026-02-06)
-------------------

1. Bug Fix: Streaming Audio Support
   修复问题：流媒体音频支持

   - Automatic detection of streaming sources using ffprobe (duration=0 or >24 hours)
   - Skip cache for streaming sources to reduce latency
   - Direct URL decoding for streaming audio

   使用 ffprobe 自动检测流媒体源（时长为0或>24小时）
   流媒体源跳过缓存以降低延迟
   流媒体音频直接 URL 解码

2. Feature: AirPlay Device Exclusion
   新功能：AirPlay 设备排除功能

   - Added AIRPLAY_EXCLUDE configuration to exclude devices by IP or name
   - Useful for filtering out unwanted AirPlay devices

   添加 AIRPLAY_EXCLUDE 配置，可按 IP 或名称排除设备
   用于过滤不需要的 AirPlay 设备

3. Bug Fix: Fixed missing XML metadata with ffprobe parsing
   修复问题：修复 ffprobe 解析时缺失 XML 元数据的问题

   - Improved metadata extraction from DLNA requests
   - Better handling of malformed XML metadata

   改进从 DLNA 请求中提取元数据
   更好地处理格式错误的 XML 元数据

4. Bug Fix: Fixed errors when switching between multiple DLNA clients during playback download
   修复问题：修复播放下载期间在多个 DLNA 客户端之间切换时的错误

   - Proper cleanup of download threads when switching clients
   - Prevent race conditions during client handover

   切换客户端时正确清理下载线程
   防止客户端切换期间的竞态条件

5. Cleanup: Removed old config file format compatibility handling
   清理：移除旧配置文件格式兼容性处理

   - Simplified configuration loading logic
   - Removed legacy format support

   简化配置加载逻辑
   移除旧格式支持


v1.1.0 (2026-02-04)
-------------------

1. Project Rename: DLNA-To-Airplay -> unAirplay
   项目重命名：DLNA-To-Airplay -> unAirplay

2. Web UI Redesign: More minimalist design with integrated DSP settings
   页面重新设计：更简约的设计，DSP 设置一体化更强

3. DSP Architecture Redesign: EQ & Spectral enhancement now support IIR/FFT/FIR modes.
   Spectral gain range expanded from 1.0-2.0 to 0.5-2.0
   DSP 架构重新设计：EQ 和频谱增强现在支持 IIR/FFT/FIR 三种模式。
   频谱增强调整范围从 1.0-2.0 扩展至 0.5-2.0

4. Cache System: Decoupled download and playback to ensure continuous playback
   even when stream URL is interrupted
   添加缓存系统：分离下载与播放操作，保证音乐 URL 断流情况下仍能继续播放

5. Bug Fix: Fixed issue where UI still shows "playing" status after track completion
   修复问题：修复歌曲播放完成后页面仍显示播放中的问题

6. UI Enhancement: Playing devices now prioritized at the top of device list
   界面优化：播放中的设备优先排列在设备列表顶部


v1.0.0 (Initial Release)
------------------------

1. DLNA to AirPlay: Complete basic playback functionality from DLNA to AirPlay devices
   DLNA 到 AirPlay：完整的从 DLNA 到 AirPlay 设备的基本播放功能

2. DLNA to Server Speaker: Complete local audio output functionality
   DLNA 到本地扬声器：完整的从 DLNA 到 Server Speaker 本地设备播放功能

3. DLNA Protocol: Standard DLNA request/response flow with optimizations for
   popular Chinese music apps (NetEase Cloud Music, QQ Music, Kugou Music,
   Kuwo Music, Migu Music)
   DLNA 协议：完成标准 DLNA 流程（请求、响应），并对中国常用音乐软件
   （网易云音乐、QQ 音乐、酷狗音乐、酷我音乐、咪咕音乐）作出 DLNA 适配

4. DSP Features: Implemented EQ, Spectral Enhancement (IIR/FFT),
   Dynamic Range Compression, and Stereo Widening
   DSP 功能：实现 EQ、频谱增强（IIR/FFT）、动态范围压缩、立体声拓宽
